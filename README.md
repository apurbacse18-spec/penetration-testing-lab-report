# Penetration Testing Lab Report

## Overview
This repository contains a sanitised penetration testing lab report completed in a controlled academic lab environment. The project demonstrates practical penetration testing methodology, vulnerability research, network enumeration, service analysis, web application testing, evidence collection, and remediation reporting.

## Project Objectives
- Conduct vulnerability research using CVE and CWE references
- Perform Linux build review and configuration analysis
- Identify live hosts and exposed services in a controlled lab network
- Analyse security weaknesses across network and web services
- Document findings with severity, impact, evidence, and remediation
- Produce a professional penetration testing report

## Key Skills Demonstrated
- Vulnerability assessment
- Network enumeration
- Service discovery
- SMB enumeration
- Web application testing
- Directory brute-forcing
- HTTP header analysis
- TLS/SSL weakness identification
- Technical report writing
- Risk-based remediation planning

## Tools Used
- Kali Linux
- Nmap
- Gobuster
- Burp Suite
- smbclient
- enum4linux
- curl
- HTTPie
- Manual web application testing

## Main Security Areas Covered
- Outdated web technologies
- Anonymous SMB read/write access
- Exposed administrative applications
- Directory enumeration
- HTTP information disclosure
- Unsupported TLS/SSL configuration
- Insecure Telnet exposure
- Web parameter manipulation

## Summary
The assessment identified multiple security issues within a controlled lab environment. Findings were categorised by severity and supported with technical evidence. Remediation recommendations focused on access control, service hardening, software updates, secure configuration, and reducing unnecessary exposure.

## Ethical and Academic Note
This project was completed in a controlled academic lab environment for educational purposes only. Sensitive personal, academic, and environment-specific details have been removed or sanitised before public upload.
